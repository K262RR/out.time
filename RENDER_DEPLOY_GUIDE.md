# 🚀 Руководство по деплою на Render.com

## Проблема и решение

**Ошибка:** `getaddrinfo ENOTFOUND base`

**Причина:** Неправильная конфигурация в `render.yaml` и попытка запуска миграций во время установки пакетов, когда база данных еще не готова.

**Решения, которые были применены:**

### 1. Исправлен render.yaml
- Изменено имя базы данных с `outtime-db` на `outtime-database`
- Обновлены команды сборки для правильной работы с monorepo структурой
- Добавлена установка корневых зависимостей перед сборкой отдельных частей

### 2. Изменен подход к миграциям
- Убран автоматический запуск миграций из `postinstall` скрипта
- Миграции теперь запускаются при старте сервера, когда база данных уже готова
- Добавлена проверка подключения к базе данных перед запуском миграций

### 3. Добавлена безопасность
- Создан скрипт `check-db-connection.js` для проверки подключения к БД
- Добавлена обработка различных типов ошибок подключения
- Улучшено логирование для диагностики проблем

## Инструкции по деплою

### Шаг 1: Подготовка проекта
Убедитесь, что все изменения зафиксированы в git:
```bash
git add .
git commit -m "Fix render deployment configuration"
git push origin main
```

### Шаг 2: Создание сервисов на Render

1. **Создайте PostgreSQL базу данных:**
   - Зайдите на render.com
   - Нажмите "New" → "PostgreSQL"
   - Имя: `outtime-database`
   - База данных: `outtime_prod`
   - Пользователь: `outtime_user`
   - План: Free

2. **Создайте Backend сервис:**
   - Нажмите "New" → "Web Service"
   - Подключите ваш GitHub репозиторий
   - Имя: `outtime-backend`
   - Среда: Node
   - Команда сборки: `npm install && cd backend && npm install`
   - Команда запуска: `cd backend && npm start`
   - План: Free

3. **Настройте переменные окружения для Backend:**
   ```
   NODE_ENV=production
   PORT=10000
   DATABASE_URL=[Автоматически из базы данных]
   JWT_SECRET=[Сгенерируйте случайную строку]
   BOT_TOKEN=[Ваш токен Telegram бота]
   API_BASE_URL=https://outtime-backend.onrender.com
   FRONTEND_URL=https://outtime-frontend.onrender.com
   ```

4. **Создайте Frontend сервис:**
   - Нажмите "New" → "Static Site"
   - Подключите ваш GitHub репозиторий
   - Имя: `outtime-frontend`
   - Команда сборки: `npm install && cd frontend && npm install && npm run build`
   - Папка публикации: `frontend/dist`

### Шаг 3: Альтернативный способ - использование render.yaml

Если хотите использовать автоматический деплой через `render.yaml`:

1. Убедитесь, что файл `render.yaml` в корне проекта обновлен
2. В Render панели выберите "New" → "Blueprint"
3. Подключите ваш репозиторий
4. Render автоматически создаст все сервисы по конфигурации

### Шаг 4: Настройка переменных окружения

Обязательно установите эти переменные:

**Backend:**
- `BOT_TOKEN` - получите у @BotFather в Telegram
- `JWT_SECRET` - сгенерируйте длинную случайную строку
- `DATABASE_URL` - будет установлена автоматически
- `FRONTEND_URL` - URL вашего фронтенда на Render

### Шаг 5: Проверка деплоя

После деплоя проверьте:
1. Логи backend сервиса - должны показать успешное подключение к БД и запуск миграций
2. Healthcheck endpoint: `https://your-backend-url.onrender.com/health`
3. API документация: `https://your-backend-url.onrender.com/api-docs`

## Диагностика проблем

### Если деплой все еще падает:

1. **Проверьте логи:**
   ```bash
   # В панели Render перейдите к сервису и откройте вкладку Logs
   ```

2. **Распространенные ошибки:**
   - `ENOTFOUND` - проблема с DNS, проверьте DATABASE_URL
   - `ECONNREFUSED` - база данных не запущена или неправильный порт
   - `28P01` - ошибка аутентификации, проверьте логин/пароль в DATABASE_URL

3. **Проверьте переменные окружения:**
   - Все обязательные переменные установлены
   - DATABASE_URL корректно ссылается на созданную базу данных
   - BOT_TOKEN действителен

### Полезные команды для локальной отладки:

```bash
# Проверка подключения к БД
cd backend && node check-db-connection.js

# Запуск миграций вручную
cd backend && npm run migrate

# Локальный запуск
cd backend && npm run dev
```

## Важные моменты

1. **База данных должна быть создана ПЕРВОЙ** перед backend сервисом
2. **Миграции выполняются автоматически** при запуске сервера
3. **Free план Render** засыпает через 15 минут бездействия
4. **Первый запуск** может занять несколько минут

## Что изменилось в коде

1. `render.yaml` - исправлены имена сервисов, команды сборки и добавлен `rootDir`
2. `backend/package.json` - убран автоматический запуск миграций из postinstall, обновлен start скрипт
3. `backend/server.js` - добавлены проверка БД и запуск миграций при старте (для разработки)
4. `backend/server.production.js` - упрощенная версия для продакшена без критических проверок
5. `backend/check-db-connection.js` - новый скрипт для проверки подключения к БД

### Последние исправления:
- Создан `server.production.js` - более устойчивая версия для продакшена
- Исправлена функция `checkDatabaseConnection` чтобы не вызывать `process.exit()`
- Обновлены команды в `render.yaml` с использованием `rootDir` для каждого сервиса
- npm start теперь запускает `server.production.js` в продакшене
